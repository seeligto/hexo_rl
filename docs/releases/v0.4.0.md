# Release v0.4.0 — Q13 chain-length planes

**Branch:** `feat/q13-chain-planes`
**Head commit:** `cd28431`
**Date:** 2026-04-15
**Type:** architectural break (minor version bump)

---

## Summary

Ships the **18 → 24 input plane** expansion. Adds 6 Q13 chain-length planes
(per-hex-axis post-placement run counts), a new chain-prediction auxiliary head,
and the Q19 threat-head `pos_weight=59` fix. Bundles the regenerated 24-plane
bootstrap model and corpus so existing users can step across the break in one
operation.

Architectural breaks:

- NN input tensor `(K, 18, 19, 19)` → `(K, 24, 19, 19)`. Planes 0–17 unchanged;
  planes 18–23 = chain-length signal per hex axis.
- Chain-prediction aux head trained against `input[:, 18:24]` (smooth_l1, weight 1.0).
- Threat-head BCE now uses `pos_weight=59` (Q19 class-imbalance fix), active
  during self-play only.
- Rust replay buffer HEXB format `v2` → `v3` (explicit `n_planes: u32` header).
- Rust `apply_symmetry_24plane<T>` kernel extracted from `ReplayBuffer::apply_sym`
  and exposed via PyO3 (`engine.apply_symmetry`, `apply_symmetries_batch`,
  `compute_chain_planes`). Pretrain augmentation now routes through this kernel
  — eliminating the `_apply_hex_sym` path that silently corrupted chain planes
  in ~92% of samples (F1 root cause, §93).

See sprint log §92 + §93 for the full design + audit trail.

## Breaking changes

### 1. Replay buffer rejected

HEXB format version `2` → `3`. Any `replay_buffer.bin` saved by a pre-`v0.4.0`
build is rejected on load with an explicit error. **Action:** archive the old
buffer; the new self-play loop rebuilds the buffer from scratch.

### 2. Model checkpoints incompatible

`input_conv.weight` shape changes from `(128, 18, 3, 3)` to `(128, 24, 3, 3)`,
and the new `chain_head` weights have no counterpart in pre-`v0.4.0`
checkpoints. `strict=True` load fails. **Action:** start self-play from the
shipped `bootstrap_model.pt`; no resume path across the break.

### 3. Corpus NPZ plane count 18 → 24

`data/bootstrap_corpus.npz` stores states as `(N, 24, 19, 19) float16`. Pre-
`v0.4.0` corpus NPZs load with a clear shape-mismatch error. **Action:** pull
the shipped `bootstrap_corpus.npz` from release assets; regenerating from
`corpus/` is optional (the NPZ here is the exact one v3b pretrain consumed).

### 4. Probe baseline schema v2 → v4

`fixtures/threat_probe_baseline.json` is re-keyed against the new bootstrap;
`BASELINE_SCHEMA_VERSION` bumped 3 → 4 after §93 pretrain v3b. **Action:**
the file ships with the release; no user action unless you maintain a custom
fixture set.

## Migration path

For users with an existing clone on `master`:

```bash
# 1. Archive old artifacts locally (out of git)
mkdir -p checkpoints/archive_18plane_pre_q13
mv checkpoints/checkpoint_*.pt \
   checkpoints/checkpoint_log.json \
   checkpoints/inference_only.pt \
   checkpoints/best_model.pt \
   checkpoints/replay_buffer.bin \
   checkpoints/archive_18plane_pre_q13/ 2>/dev/null || true

# 2. Pull the v0.4.0 branch / tag
git fetch origin
git checkout v0.4.0   # or: git checkout master && git pull

# 3. Rebuild the Rust extension (N_PLANES, apply_sym kernel changed)
make rebuild

# 4. Download the new bootstrap + corpus from the release assets
bash scripts/install.sh   # re-runs download_and_verify against v0.4.0 hashes
# OR manually from https://github.com/seeligto/hexo_rl/releases/tag/v0.4.0 :
#   checkpoints/bootstrap_model.pt
#   data/bootstrap_corpus.npz
#   fixtures/threat_probe_baseline.json

# 5. Verify
make test             # 746 python + 4 rust pass
make probe.bootstrap  # writes v4 baseline, exits 0

# 6. Start self-play on the v3b bootstrap
make train
```

**Re-running `make pretrain` is NOT required** — the shipped `bootstrap_model.pt`
IS the v3b artifact from §93 C16. Re-running pretrain would produce a
byte-different model (different RNG state) without changing the downgraded
expectations. Ship binaries, skip the ~40 min pretrain on user hardware.

## Benchmark movement (2026-04-15, n=5 median)

Run on the same RTX 3070 desktop as the pretrain. Numbers reflect the
+33% state-byte cost of 24-plane states plus the plane-aware `axis_perm`
scatter path added in C3.

| Metric | Pre-Q13 target | v0.4.0 measured | Verdict |
|---|---|---|---|
| MCTS sim/s (CPU, no NN) | ≥ 26,000 | 54,911 | PASS |
| NN inference batch=64 pos/s | ≥ 8,250 | 8,349 | PASS |
| NN latency batch=1 mean ms | ≤ 3.5 | 1.73 | PASS |
| GPU utilisation % | ≥ 85 | 100.0 | PASS |
| Worker pool throughput pos/hr | ≥ 500,000 | 522,425 | PASS |
| Worker batch fill % | ≥ 80 | 100.0 | PASS |
| **Buffer push pos/s** | ≥ 630,000 | 574,585 | **FAIL (−8.7%)** |
| **Buffer sample augmented µs/batch** | ≤ 1,400 | 1,638 | **FAIL (+17%)** |
| Buffer sample raw µs/batch | ≤ 1,500 | 1,464 | PASS (barely) |

Both failing metrics are **structural regressions from the 18 → 24 plane
break**, not code defects. Per §92 the planned movers were buffer sample
augmented, NN forward latency batch=1, and NN inference batch=64 — the first
landed as predicted; NN latency came in well under target (1.73 ms vs 3.5 ms
ceiling); NN inference cleared 8,250 pos/s by ~1%. The unplanned mover is
buffer push, where IQR is ±34% (407k–696k) — likely a cache-residency / arena
effect that will stabilise post-rebaseline.

Follow-up after merge (separate commit): rebase the CLAUDE.md benchmark table
to reflect 24-plane floors — `buffer push ≥ 500,000 pos/s` and
`buffer sample augmented ≤ 1,750 µs`. No architectural budget breached.

## Pretrain v3b results (from §93 C16)

15 epochs × 779 batches × 256 batch_size = 11,685 steps on RTX 3070
(~39.5 min wall time). Rust apply_symmetries_batch path used throughout.

| metric | gate | v3 (broken aug) | v3b (fixed) |
|---|---|---|---|
| policy_loss (final) | ≤ 2.47 | 2.1758 | **2.1758** |
| value_loss (final)  | ≤ 0.59 | 0.5021 | **0.4990** |
| opp_reply_loss      | — | 2.1879 | **2.1846** |
| chain_loss (final)  | ≤ 0.01 (Q21 degenerate slice) | 0.0019 | **0.0018** |
| 100-game RandomBot greedy wins | ≥ 95 | 100/100 | **100/100** |

The v3 vs v3b loss-scalar delta is near-zero because the aux-head target is a
slice of the input — degenerate identity mapping. The win is that the 6
per-axis chain planes the trunk consumes are now byte-exactly consistent with
the stones under all 12 symmetries (v3 was cross-axis garbage in ~92% of
augmented samples). Q13 uplift is expected to manifest during sustained
self-play, not in the pretrain scalar.

## Expectations (downgraded from KataGo 1.65×)

The review literature cites KataGo's 1.65× tactical-probe acceleration from
auxiliary **future-information** targets (terminal ownership). Our chain-aux
target is a slice of the **current** input — it provides regularization,
intermediate supervision, and preservation pressure on chain-counting
circuits, but not counterfactual forward information. Realistic uplift is
**1.1–1.3× on tactical probe convergence**, plus faster open-4 detection
from the explicit per-axis chain signal.

Q21 (wider-window chain-aux target) is the genuine forward-information
variant and remains parked post-baseline.
