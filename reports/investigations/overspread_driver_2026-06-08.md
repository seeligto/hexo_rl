## Operator insight folded in — TURN vs PLY (the unit of "winning")

A turn places **2 stones** (1 in the opening). The Rust `count_winning_moves` and the MCTS
quiescence value-override (`engine/src/mcts/backup.rs`: `count_winning_moves(p) >= 3` → +1) are
**depth-1** (single-stone) — the wrong unit for "can I finish *this* turn." The turn-correct
primitive (`scripts/structural_diagnosis/turn_wins.py`):

```
winning_turn_cells(b, side) = { depth1 completions } ∪ { second stone of each depth2 completion }
count_winning_turns(b, side) = |winning_turn_cells|     # turn-aware via depth2's moves_remaining guard
```

**Empirics** (`turn_wins.py` self-test, 3069 banked threat snapshots, 0 invariant violations):
depth-1 **undercounts the turn win-set in 86.5%** of threat snapshots. At in-window forced-win
**turn-starts** (`overspread_forkredundancy.py`): the wins are ~95% depth-2; depth-1 sees nothing
(`ply_blind` 0.93–0.98) and the engine quiescence **`credit_gap = 1.000`** — it *never* fires its
+1 forced-win override at these positions. **Strength order:** this-turn depth-2 completion (a
single one = forced win) **>** next-turn depth-1 ≥3 fork **>** single ply threat. The engine credits
only the **weaker** (depth-1 ≥3) and is **blind to the stronger** (depth-2 this-turn) — an
inversion, not just an undercount. This gap is **flat across the arc** (a static structural fact),
so it is the standing *condition*, not the trend.

The turn-correct primitive is used throughout D1 (concentration label) and D3 (fork-affinity).
**Recommend promoting `count_winning_turns` into `forced_win_detector.py`** (unifying the existing
f-vs-s inconsistency across the §D-COHERENCE scripts) as a deliberate follow-up. The depth-1
quiescence limitation is **eval-MEASURED, not changed** (Phase-B, perf-sensitive: depth-2 is
O(threats²)/leaf, deliberately omitted §28/§30). `probe_threat_logits.py` flagged for a
turn-vs-ply audit.

## Phase A — the five drivers (eval-only, pre-registered)

### D1 — VALUE-DISCRIMINATION CEILING (concentration framing) → **OUT**
Pool: 856 terminal-WON in-window forced-win snapshots (30k/53k/87.5k), **outcome held constant**
so this measures concentration-ranking, NOT the banked outcome-AUC=0.79. AUC_conc(mover_ncomp,
median split) per 11 checkpoints: 0.640–0.752, **mean 0.675, every CI above 0.60**; tertile-extreme
0.701. **Red-team correction (load-bearing):** the headline mover_ncomp + largest_blob_frac strands
are **stone-confounded** (corr(ncomp,stones)=0.75; within-stone-band ncomp AUC softens to ~0.55,
largest_frac to 0.58). The **only confound-free strand** is the **turn-correct fork-redundancy**
AUC (`count_winning_turns≥3`-won > `==1`-won): corr-to-stones 0.166 (clean), **stone-matched
0.69–0.79, min 0.693, RISING** 30k→90k. → Value not only sees concentration, it ranks turn-fork
redundancy **increasingly well** over the arc. LIT needs ≤0.60 AND declining; observed >0.60 AND
rising → **OUT**. §2 guard: did not reuse the banked 0.79; tested fresh (transfers=no). *Value is a
clean signal the fix can lean ON, not a hole to patch.*

### D2 — OFF-WINDOW STRUCTURE BIASES PLAY (spread driver) → **OUT**

abandonment gap **negative** every step (boundary abandoned *less*); (3) decisive conditional

The lone D2-shaped signal (boundary-share 0.085→0.199) is a **symptom** of over-spread (force
fragments outward toward the frontier), arrow named. → **OUT**. §2 guard: V-OFFWINDOW refutation is

## Output to the strategy layer

**Causal picture (best supported, with honesty about what is unproven):**
- Over-spread is **NOT** caused by a value ceiling (D1 OUT — value ranks concentration, rising),
  **NOT** by missing target fork-credit (D3 OUT — target credits forks ~180× null), **NOT** by
  off-window avoidance (D2 OUT — model commits to the boundary), **NOT** by exploration drift
  (D4 OUT — constant).
- It is generated by the **self-play dynamic upstream of value/search**. The proximate trend
  mechanism is unidentified-but-self-play-flavored; **D5 (spread-vs-spread, never punished
  internally) is the leading candidate but INCONCLUSIVE** (instrument-blocked + non-specific
  correlation).
- The **standing structural hole** that lets the spread go uncorrected is the **turn-vs-ply
  depth-1 credit gap**: no *hard* internal credit for turn-fork concentration anywhere in the
  value/quiescence/target path (credit_gap=1.0); the only anti-spread signal is the *soft* NN
  value ranking, which is too weak to break the spread-vs-spread equilibrium.

**Routed fix (NOT value-first; two complementary internal-pressure branches; each carries a §2
guard; ALL gated behind operator + a 30k SIGNATURE smoke):**

1. **Close the standing credit-gap hole — turn-correct hard concentration/fork credit.** Promote
   `count_winning_turns` into the value/target credit path (quiescence override and/or an auxiliary
   value target predicting turn-fork-redundancy) so a thin win and a concentrated turn-fork no
   longer receive the same value — a *hard* gradient toward concentration. This unifies the
   dispatcher's D1-fix (aux concentration target) + D3-fix (fork-redundancy target shaping) via the
   turn-correct primitive, **re-motivated as closing the structural hole, not patching a broken
   head** (heads aren't broken). GUARDS: not the falsified A2 avg-pool (target-side, not
   aggregation-arch); not a config knob (target/value shaping); the quiescence variant is
   perf-sensitive → **bench-gate** (depth-2 O(threats²)/leaf). Cheaper first test: fine-tune the
   value head on the turn-fork-redundancy aux target on a banked checkpoint.
2. **D5 compact-reference self-play regularizer — the co-adaptation backstop.** A compact opponent
   or regularizer that punishes spread INTERNALLY so spread-vs-spread self-play gains a compact
   counter-pressure. GUARD: bot-mix was falsified as an *anti-colony* config knob — this is an
   over-spread-correction regularizer (opposite pole, different mechanism class); NOT a full PFSP
   league.

**Recommended SEQUENCE + the decisive missing instrument.** D1/D3 OUT means value/target *already*
